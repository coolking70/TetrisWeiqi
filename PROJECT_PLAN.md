# 俄罗斯方块棋 - 完整项目规划

## 一、核心玩法可行性评估

### 当前状态
- 10x10 棋盘，7种标准俄罗斯方块
- 围棋围杀 + 俄罗斯方块消行双重机制
- 基础启发式AI（三级难度）

### 游戏复杂度分析（与围棋对比）

| 维度 | 围棋(19x19) | 俄罗斯方块棋(10x10) |
|------|------------|-------------------|
| 棋盘格数 | 361 | 100 |
| 每步动作空间 | ~361 | ~2800（7块×4旋转×100位置，实际有效≈200-800） |
| 状态空间 | ~10^170 | ~10^70（估算，远小于围棋） |
| 平均对局步数 | ~200 | ~40-80 |
| 随机性 | 无 | 有（随机方块）|
| 信息完整性 | 完全信息 | 不完全信息（未来方块未知）|

**结论：复杂度远低于围棋，AI训练完全可行，且对硬件要求友好。**

## 二、AI 训练方案

### 方案对比

#### 方案A：AlphaZero-Lite（推荐起步方案）
```
架构：CNN/ResNet(6-10层) + MCTS
输入：10×10×5 棋盘状态 + 当前方块编码
输出：策略头(走法概率) + 价值头(胜率)
训练：自我对弈 → 收集棋谱 → 训练网络 → 循环
```
- **硬件需求**：单张 RTX 3060 (12GB) 即可，M1/M2 Mac 也能跑
- **训练时间**：预计 3-7 天可达到超越人类业余水平
- **优势**：成熟方案，效果好，可解释性强
- **框架**：PyTorch + 自定义MCTS

#### 方案B：PPO 强化学习（最快出结果）
```
架构：Actor-Critic，较浅的全连接/CNN网络
训练：环境交互 → 策略梯度更新
```
- **硬件需求**：CPU 即可训练，GPU 加速明显
- **训练时间**：1-3 天
- **优势**：实现简单，迭代快
- **劣势**：棋力上限低于MCTS方案
- **框架**：Stable-Baselines3 / RLlib

#### 方案C：MuZero-Lite（高阶方案）
```
架构：学习环境动态模型 + MCTS
无需游戏规则硬编码到搜索中
```
- **硬件需求**：RTX 3080+ (16GB+) 或 M2 Pro+
- **训练时间**：1-2 周
- **优势**：处理随机方块更优雅，泛化能力强
- **劣势**：实现复杂度高

#### 方案D：进化策略 + 启发式（最低门槛）
```
用遗传算法优化当前启发式AI的权重参数
无需GPU，纯CPU可训练
```
- **硬件需求**：任意电脑
- **训练时间**：数小时
- **优势**：零门槛，与现有代码无缝集成
- **劣势**：棋力有限，没有深度策略

### 推荐路线：渐进式升级
```
第1阶段：方案D（进化策略优化现有AI） → 立即可做
第2阶段：方案B（PPO训练基础RL模型） → 1-2周
第3阶段：方案A（AlphaZero-Lite + MCTS）→ 1-2月
第4阶段：方案C（按需决定是否升级）
```

### AI推理部署方案
- **浏览器端**：用 ONNX.js / TensorFlow.js 将训练好的模型转为浏览器可运行格式
- **模型体积**：6-10层 ResNet 量化后约 2-5MB，完全可嵌入网页
- **推理速度**：单次决策 < 100ms（浏览器端），实时交互无延迟
- **离线可用**：模型内嵌，无需服务器

## 三、RPG 融合设计方案

### 3.1 世界观框架

**"棋域大陆"** —— 一个由方块构成的世界，两大势力（蓝晶/紫焰）围绕"棋域"展开争夺。玩家作为新晋棋术师，通过对弈成长，揭开棋域的秘密。

### 3.2 核心系统设计

#### 角色系统
```
6个职业 × 不同能力：

┌─────────┬─────────────────────────┬──────────────────────┐
│ 职业     │ 被动能力               │ 主动技能(消耗能量)    │
├─────────┼─────────────────────────┼──────────────────────┤
│ 建筑师   │ L/J块出现概率+15%      │ 将任意方块变为L块     │
│ 破坏者   │ 围杀额外产生1格死区     │ 炸毁3x3区域          │
│ 先知     │ 可预览后2个方块         │ 重新抽取当前方块      │
│ 术士     │ 死亡区域可被己方穿越     │ 将敌方1个方块变为己方  │
│ 工程师   │ 消行时保留己方方块      │ 旋转棋盘90°          │
│ 时空者   │ 每3回合额外一次落子     │ 撤回对方上一步        │
└─────────┴─────────────────────────┴──────────────────────┘
```

#### 成长系统
```
经验值 → 等级提升 → 技能点分配

技能树（每职业3条分支）：
├── 攻击系：增强围杀、死区扩展
├── 防御系：自动气眼、方块护盾
└── 辅助系：方块预览、选择、变形
```

#### 装备系统
```
6个装备槽：头/身/手/脚/饰品/棋盘皮肤

装备效果举例：
- "先手之冠"：50%概率先手
- "融合手套"：可将两个方块合并放置（消耗2回合）
- "镜像靴"：可镜像翻转方块
- "虚空戒指"：死亡区域每5回合消失1格
```

### 3.3 游戏模式

#### 单人内容
1. **剧情战役**（30+关卡）
   - 线性主线 + 分支支线
   - Boss战：AI拥有特殊规则（如Boss每回合放2块）
   - 解锁角色、装备、棋盘

2. **挑战模式**
   - 限时挑战：30秒内落子
   - 残局模式：给定局面，N步内围杀
   - 每日挑战：随机规则变体

3. **AI对弈**
   - 自由对弈（选择AI强度）
   - AI复盘分析（标注好手/疑问手）
   - AI教学模式（提示最佳落点）

#### 多人内容
4. **排位对战**
   - 段位系统：青铜→白银→黄金→铂金→钻石→大师→宗师
   - ELO 匹配

5. **锦标赛**
   - 8人/16人淘汰赛
   - 赛季制

6. **合作模式**
   - 2v2：队友共享棋盘，交替落子
   - Boss挑战：多人协力对抗超强AI

### 3.4 棋盘变体系统
```
不同地图/棋盘增加策略深度：

- 标准棋盘：10×10 经典
- 十字棋盘：中间交叉区域计双倍分
- 迷雾棋盘：只能看到己方周围3格
- 障碍棋盘：预设不可放置区域
- 传送棋盘：边缘连通（上下/左右相连）
- 扩展棋盘：每10回合棋盘扩大1圈
```

## 四、技术架构规划

### 4.1 整体架构
```
┌─────────────────────────────────────────────────┐
│                   客户端                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ 游戏引擎  │ │ UI/UX    │ │ AI推理(ONNX.js) │ │
│  │ (Phaser/  │ │ (Vue/    │ │ 浏览器端运行     │ │
│  │  PixiJS)  │ │  React)  │ │                  │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
├─────────────────────────────────────────────────┤
│                   服务端                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ 对战服务  │ │ 用户系统  │ │ AI训练Pipeline  │ │
│  │ WebSocket │ │ 账号/存档 │ │ PyTorch + MCTS  │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
├─────────────────────────────────────────────────┤
│                   数据层                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ MongoDB   │ │ Redis    │ │ 棋谱数据库       │ │
│  │ 用户/存档 │ │ 排行/匹配│ │ 训练数据         │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
└─────────────────────────────────────────────────┘
```

### 4.2 技术选型

| 模块 | 推荐方案 | 备选 |
|------|---------|------|
| 游戏引擎 | Phaser 3 (2D) | PixiJS / Godot(跨平台) |
| 前端框架 | Vue 3 + Vite | React |
| 后端 | Node.js + Express | Go / Python FastAPI |
| 实时通信 | Socket.io | WebSocket原生 |
| AI训练 | PyTorch | JAX |
| AI部署 | ONNX Runtime Web | TensorFlow.js |
| 数据库 | SQLite(单机) → PostgreSQL(联网) | MongoDB |
| 桌面打包 | Electron / Tauri | — |
| 移动端 | PWA → Capacitor | React Native |

### 4.3 单机优先策略
考虑到个人开发者的资源限制，建议**先做单机版**：
- 所有数据存 localStorage / IndexedDB
- AI模型内嵌在前端
- 无需服务器即可完整游戏
- 后续再加联网功能

## 六、硬件需求总结

### 最低配置（开发+训练基础AI）
- CPU: 任意现代多核处理器
- RAM: 8GB
- GPU: 无需独显（CPU训练PPO）
- 存储: 10GB

### 推荐配置（训练AlphaZero-Lite）
- CPU: 8核+
- RAM: 16GB
- GPU: RTX 3060 12GB 或 M1/M2 Mac 16GB
- 存储: 50GB SSD

### 理想配置（快速迭代训练）
- GPU: RTX 4070+ / M2 Pro+
- RAM: 32GB
- 可考虑云GPU（Colab Pro、AutoDL等按时计费）
